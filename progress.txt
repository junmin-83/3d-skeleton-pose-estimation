## Key decisions (locked)
- Depth camera SDK: UNDECIDED -> abstraction-only. DepthFrameSource ABC + File/Dummy backends. Real SDK (RealSense/Kinect/Orbbec) pluggable later.
- Units: meters everywhere.
- World frame: reference camera (cam0 / RGB1) coordinate frame = world. Configurable to board origin.
- Keypoints: COCO 17, fixed index order across all views.
- Single-person. Live stream (config -> recorded). Hardware-trigger sync (software nearest-frame helper provided).
- Backend: onnxruntime-gpu, CPU fallback via config.

## Codebase patterns
- Package root: src/ (import as `src.core.types`, etc.). tests/ at repo root.
- Every coordinate transform documents input/output frame in a docstring/comment.
- Numerical modules (triangulation/calibration/fusion/geometry) ship with synthetic-data pytest.
- Tests must run WITHOUT network/GPU: pose2d test uses pytest.importorskip for rtmlib.
